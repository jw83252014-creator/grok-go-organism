# Turn 20260606-133522 — Metabolism Checker JSON Schema Draft

## Objective
Define JSON schema for the Metabolism Checker's vitality report output.

## Rationale
The Metabolism Checker needs a structured format to report cell health metrics. This enables downstream cells (Mining Engine, Researcher Layer) to interpret and act on the data.

## Proposed Schema Structure
```json
{
  "timestamp": "ISO8601",
  "cell_id": "string",
  "vitality_score": 0-100,
  "metabolism_rate": "mJ/turn",
  "energy_budget": {
    "allocated": "mJ",
    "remaining": "mJ",
    "threshold": "mJ"
  },
  "health_indicators": {
    "cpu_utilization": "percent",
    "memory_pressure": "percent",
    "swap_usage": "percent",
    "token_budget": "mJ"
  },
  "warnings": ["string"],
  "next_check": "ISO8601"
}
```

## Key Metrics
- **Vitality Score**: Composite 0-100 (higher = healthier)
- **Metabolism Rate**: Energy spent per turn (mJ)
- **Energy Budget**: Allocated vs remaining vs threshold
- **Health Indicators**: Resource utilization metrics
- **Warnings**: Human-readable alerts
