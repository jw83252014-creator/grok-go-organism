# Grok Go Turn 20260607-160000 — Google TurboQuant Research

## Summary
Researched Google's TurboQuant KV-cache compression (ICLR 2026, PolarQuant + QJL 1-bit residual). Confirmed mechanism, performance, and integration path for Grok Go.

## Key Findings
- **Algorithm**: 2-stage vector quantization with PolarQuant (random orthogonal rotation + optimal scalar quantizer) + 1-bit QJL residual correction
- **Compression**: 6× memory reduction, 8× attention speedup on NVIDIA H100
- **Accuracy**: Quality-neutral at 3-4 bit quantization (0.957 logit cosine similarity for Qwen3-4B); 4-bit achieves 4/4 top-1 token accuracy
- **Data-oblivious**: No training required; codebooks precomputed
- **Apple Silicon**: Implemented in ~200 lines pure mlx.core (mlx-lm issue #1060); 4-bit works but decode 0.5x FP16 due to dequantize-on-fetch overhead

## Integration Path for Grok Go
1. **Local Qwen3-14B-MLX-4bit**: Direct compatibility with existing mlx-lm backend. QJL 1-bit residual correction pairs naturally with 4-bit models.
2. **Hybrid (Grok paid)**: Wire as OpenAI-compatible wrapper with KV cache compression layer. X-Routed-Via: turboquant/nvidia.
3. **Mining Engine telemetry**: Track compressed KV size, attention latency, energy savings.
