## Validation Rules Draft
```
# Mining Engine: Field Validation Rules
# Purpose: Ensure data integrity before extraction and indexing

## String Fields
- max_length: Hard cap (e.g., 256 chars for titles, 1024 for descriptions)
- trim_whitespace: Always trim
- escape_special: URL-safe encoding for links
- null_check: Reject null/empty unless optional

## Numeric Fields
- min_value: Hard lower bound (e.g., score >= 0)
- max_value: Hard upper bound (e.g., rating <= 5)
- precision: Fixed decimal places (2 for currency, 4 for timestamps)
- type_check: Reject non-numeric where numeric expected

## Date/Time Fields
- format: ISO8601 (YYYY-MM-DDTHH:MM:SSZ)
- timezone: UTC default
- valid_range: Within reasonable bounds (no future dates > 10 years)
- timezone_conversion: Handle DST transitions

## Boolean Fields
- true_only: Accept true/1/yes/t
- false_only: Accept false/0/no/f
- enum: Strict enum values

## Array Fields
- max_items: Hard cap (e.g., tags max 20, tags max 50)
- unique: Detect duplicates
- type_check: All items must match declared type
- min_items: Reject empty unless optional

## Special Fields
- id: UUID4 format (8-4-4-4-12 hex)
- email: RFC5322 compliant
- phone: E.164 format
- url: HTTP/HTTPS scheme required
- file: Valid MIME type
```

## Edge Cases Covered
1. Leading/trailing whitespace in string fields
2. Unicode characters in titles
3. Malformed UUIDs
4. Invalid ISO8601 timestamps
5. Negative scores/ratings
6. Overly long descriptions
7. Duplicate tags
8. Email without @ symbol
9. URL without scheme
10. Boolean with unexpected values
