# Mining Engine Data Ingestion Interface Spec

## Objective
Define field schemas for the data ingestion interface that extracts structured knowledge from raw data streams.

## Core Fields (from research stack map)

### 1. Source Metadata
- `source_id`: string (unique identifier for data origin)
- `source_type`: enum ["web", "api", "file", "agent"]
- `ingestion_timestamp`: ISO8601 datetime
- `hash`: SHA-256 (integrity check)

### 2. Content Metadata
- `content_type`: enum ["text", "json", "markdown", "image", "code"]
- `language`: string (for code/text content)
- `character_count`: integer
- `token_estimate`: integer (approximate)

### 3. Extraction Quality
- `confidence_score`: float 0.0-1.0
- `extraction_success`: boolean
- `partial_match`: boolean (with partial_match_reason)

### 4. Linkage
- `related_sources`: array of source_id (for cross-referencing)
- `wiki_entry_id`: string (target wiki entry, if applicable)

### 5. Processing State
- `status`: enum ["pending", "extracting", "validated", "indexed", "archived"]
- `next_processing_step`: string

## Validation Rules
- SHA-256 hash must match content
- confidence_score must be >= 0.5 for acceptance
- source_id must be globally unique
- ingestion_timestamp must be within last 24 hours (with exception flags)
