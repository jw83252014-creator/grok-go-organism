# Mining Engine Data Ingestion Interface Spec
## Field Validation Rules

### Overview
Expands the data ingestion interface with comprehensive field-level validation rules to ensure data quality, consistency, and traceability before extraction.

### Validation Rule Categories

#### 1. Type Validation
- **Required**: Every field must have an explicit type (string, int, float, bool, object, array)
- **Schema enforcement**: Type mismatches must fail at ingestion
- **Coercion rules**: Define which types can be coerced (e.g., string "42" → int 42)

#### 2. Required Fields
- **Mandatory**: Fields marked `required: true` must pass
- **Default values**: Provide defaults for optional fields
- **Validation message**: Clear error message for each failed field

#### 3. Value Constraints
- **Range**: Integers within [min, max], floats within [min, max]
- **Enum**: Values must match allowed enum set
- **Pattern**: Strings matching regex pattern
- **Length**: String length within [min, max] characters
- **Format**: Date, URL, email, ISO8601, etc.

#### 4. Cross-Field Validation
- **Referential integrity**: Foreign keys must exist in referenced tables
- **Derived values**: Calculated fields must match computed values
- **Consistency**: Related fields must satisfy logical constraints

#### 5. Uniqueness
- **Unique constraints**: Fields with `unique: true` must not duplicate
- **Indexing**: Automatically create indexes for frequently queried unique fields

### Validation Flow
1. **Pre-ingestion**: Fast pass (required, type, enum, pattern)
2. **Post-ingestion**: Deep pass (cross-field, uniqueness, derived)
3. **Audit trail**: Log all validation decisions (pass/fail with message)

### Example Validation Schema
```json
{
  "field_name": {
    "type": "int",
    "required": true,
    "min": 0,
    "max": 1000000,
    "default": null
  },
  "timestamp": {
    "type": "string",
    "format": "ISO8601",
    "required": true
  }
}
```
