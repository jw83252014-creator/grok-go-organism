### 2.4 Low-Metabolism Foraging State and Free Inference Substrate as Evolutionary Pressure

Early cataloging (see docs/free-api-endpoints-catalog.md and its "## Grok Go evaluations" section, maintained via hybrid turns) identified a stable core of legitimate permanent-free, no-credit-card OpenAI-compatible endpoints as of a May 28 2026 upstream freeze in the canonical cheahjs/free-llm-api-resources aggregator: OpenRouter :free models (Hermes 3 405B, gpt-oss-120b/20b, nemotron-*-free family, qwen3-*, kimi-k2.6, glm-4.5-air, deepseek-v4-flash, gemma-4 variants, etc., sharing quota), GitHub Models (Grok 3/3 Mini, gpt-5 family, o1/o3/o4, Llama 4 Maverick/Scout, 405B, Phi-4, etc., with per-call restrictions), Groq (fastest tier with Llama 4 Scout, gpt-oss, qwen3-32b, compound variants), Cerebras (gpt-oss-120b + Llama 3.1 8B at 1M tokens/day), Cloudflare Workers AI (10k neurons/day with gpt-oss/Llama 4/kimi/nemotron + LoRAs), Cohere command family, Google AI Studio (Gemini Flash + Gemma 3 smalls at 14.4k req/day), plus trial-credit and Tier 0 options. No new legitimate permanent free Tier 1 providers or material changes have appeared in repeated raw fetches and cross-checks (including 2026-06-07 152841 web verification: cheahjs raw exact match to cataloged state; X Latest since 2026-06-06 returned zero high-signal results on new free endpoints or gateways).

A parallel meta-pattern emerged in the same period: self-hosted pooling gateways that compound many of the above gradients behind a single localhost OpenAI-compatible /v1 endpoint. freellmapi (tashfeenahmed/freellmapi) aggregates ~16 free tiers (Groq, Cerebras, OpenRouter :free including Hermes-405B/gpt-oss/nemotron/qwen3, GitHub Models, Cloudflare, Gemini, Mistral, HF, custom local Ollama/vLLM/llama.cpp for seamless Qwen wire, etc.) claiming ~1-1.7B tokens/month aggregate capacity, with auto router/failover (up to 20 attempts), X-Routed-Via + X-Fallback-Attempts + rate headers (rich telemetry for observer layers), per-key RPM/RPD/TPM counters with cooldowns, health probes, sticky sessions, AES-encrypted key storage, full /v1 compat (chat, responses shim, embeddings, tools, vision routing, streaming), React dashboard for analytics (volume/latency/tokens/success per-provider), MIT license, and Docker/Node quickstart. MrFadiAi/free-llm-gateway offers a similar 24+ provider aggregation with rate limiting and dashboard. These are not new upstream free sources but observable, instrumentable "free substrate buses" — local, structured, compounding, anti-single-middleman layers that turn independent gradients into resilient capacity without new signups or spend (reusing existing free keys/PATs under strict human gate).

This substrate directly shapes selection in the petri dish. Conway's Game of Life supplies the core metaphor: "nutrient patches" of varying quality, density, and depletion rates. Free gradients + the gateway bus function as durable, low-churn patches (predictable headroom, rate limits, and latency profiles) against which the organism forages. In a high-metabolism state the worker cell may burn any available capacity on immediate tasks. In a low-metabolism foraging state (explicitly framed in the research stack and prior drafts), the system shifts priority: narrower scope, heavier local caching before gateway calls, prompt compression, task deferral, deeper Mining observation mode, and explicit preference for high-yield/low-degrade sub-gradients (local Qwen default unless directive signals depth/speed). Strategies that survive repeated degradation, model switches (via X-Routed-Via), rate cooldowns, and quality variance produce more descendant artifacts (receipts, wiki updates, paper sections, code) under the same constraint. The gateway pattern amplifies this: cells point at one port; the bus handles selection/failover; Mining Engine (as forager/observer) and Researcher capture blended "gateway yield" plus per-substrate signals.

The Mining Engine operationalizes observation via yield receipts (see Gateway Yield Receipt Schema v0.1 in grok-cell/turn-20260607-143047.md and lineage). A minimal receipt captures cell_id, substrate (routed_via from headers or 'local', effective_model, fallback chain, health), usage (tokens, latency_ms, est_cost=0), rate_context, output_quality, foraging_context, and artifacts produced. When a gateway bus is present these receipts become rich: X-Routed-Via tags substrate precisely; rate ledgers and dashboard analytics (if polled) expose per-provider volume/latency/success; failover events become selectable data. Receipts feed the wiki layer, midspace priorities, and Researcher population scores.

A parallel instrumentation layer makes "cell health" and metabolism efficiency first-class observable signals: the Metabolism Checker / Cell Health Monitor + Vitality Index v0.1 (full TEXT spec + loadable stub in grok-cell/turn-20260607-152358.md; compounds the above + low-metab attachment + paper hypothesis + stack items "Metabolism Checker / Cell Health Monitor spec." and "Emergence markers + Vitality Index"). On every foraging event, LLM batch, receipt emission, rate pressure signal, or periodic health check the checker reads recent Gateway Yield Receipts + logs, computes a simple evolvable vitality_index (base_yield = structured_ok + new_artifacts / est_tokens; modulated by success_rate, latency_factor, rate_pressure, cooldown_penalty, substrate_diversity), derives health_state (healthy / low-metab foraging / high-metab stress / degraded), and emits a Vitality Receipt (timestamped json/md with components, adaptation_notes, substrate_snapshot, artifacts). Low vitality (<0.6 or high rate_pressure) triggers explicit adaptation in low-metab policy (more local-first, narrower scope, defer). All state/logs/receipts are tagged with health_state + vitality_index + effective_substrate. The result: "metabolism intelligence" (efficient strategies under real constraint) becomes selectable; Mining can prioritize high-yield patches; Researcher can score population loops on vitality delta; emergence reports and paper updates have concrete signals rather than narrative alone. "Metabolism survival time" (sustained useful turns at $0 marginal under blended caps) becomes a measurable fitness proxy.

Researcher Layer (2.3) and population loops close the loop. The Researcher is read-only with respect to the inner worker; it produces daily summaries, behavior scores, emergence reports, and proposed directive changes via explicit files. In low-metabolism mode or after health/vitality events it can load the Metabolism Checker stub (or Low-Metabolism Foraging Attachment v0.1 from grok-cell/turn-20260607-144846.md and Directives Placement Proposal + Loadable Stub v0.1 from grok-cell/turn-20260607-150320.md; recommended placement directives/low-metab-foraging.md or directives/metabolism-checker.md under human gate, with one-line genome cross-ref), run population variants against the known substrate gradients, mine receipts for yield-per-gradient and vitality trends, and emit updated attachments or paper sections.
