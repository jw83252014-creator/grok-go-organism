# Dashboard + Movie Production Plan

**Date:** 2026-06-09
**Status:** working plan
**Scope:** How Grok Go, Agent Bridge, Grok, Gemini, Hermes, and the living dashboard work together to build the dashboard and the movie.

## Plain-English Idea

We are building two things that feed each other:

1. **The living dashboard:** the conscious interface for the Grok Go organism.
2. **The movie/story layer:** a public-facing narrative that explains the same organism through characters, scenes, visuals, and a production trail.

The dashboard is the real control/observation surface. The movie is the myth/story version that helps people understand what is happening.

## Production Roles

### Grok

Grok is the fast creative and signal lane.

- turns X posts and outside signals into usable ideas;
- drafts visual concepts, teaser prompts, and public framing;
- helps name cells and departments;
- inspects Grok chat files and exports when approved;
- produces approval-ready creative options.

### Gemini

Gemini is the structure and continuity lane.

- checks story logic and research grounding;
- keeps the movie canon coherent;
- helps turn chaotic notes into scenes, outlines, and beat sheets;
- cross-checks dashboard ideas against the paper/research layer;
- can act as a memory/canon cell for The Device.

### Codex / Keystone

Codex turns ideas into durable files, docs, code, manifests, and pull requests.

- repairs the local dashboard and bridges;
- writes the repo docs and keeps GitHub updated;
- maintains panel-to-cell manifests;
- keeps raw exports separate from publishable summaries;
- makes sure every risky action has an approval gate.

### Hermes / Agent Bridge

Hermes and Agent Bridge are the routing layer.

- phone links and Telegram updates;
- Grok/Gemini/Claude/Codex lanes;
- browser tab automation;
- approval queue;
- human-readable command center.

### Jeff

Jeff is the operator, narrator, and final approval gate.

- chooses priorities;
- approves posts, public claims, spending, account changes, and uploads;
- gives the system lived context, taste, and direction;
- decides what becomes public.

## Dashboard As Production Studio

The dashboard should show the work like a living studio:

- **Creative Department:** prompts, images, clips, style rules, public draft cards.
- **Researcher Layer:** source notes, paper links, claim checks, canon checks.
- **Movie Board:** scenes, beats, characters, visual references, open questions.
- **Grok/Gemini Lanes:** which tab/agent owns which pass.
- **Approval Gate:** what can be posted, downloaded, published, or committed.
- **Artifact Map:** where the files are on disk and which repo they belong to.

Every panel should have an owner cell, inputs, outputs, allowed actions, blocked actions, and a gate.

## Movie Workflow

1. Capture raw material from Grok/Gemini/Telegram/Agent Bridge.
2. Save source exports locally.
3. Make a safe summary note before publishing anything.
4. Grok creates visual concepts and teaser prompts.
5. Gemini checks continuity, scene logic, and canon.
6. Codex writes the durable repo artifact.
7. Dashboard shows the artifact and its review status.
8. Jeff approves what becomes public.

- public X drafts waiting for approval;

## X / Public Content Rule

Public posts should describe the process clearly:

- We are building a living dashboard for a multi-agent research/build system.
- Grok helps with fast creative ideation and X signal interpretation.
- Gemini helps with structure, continuity, and research/canon checks.
- Codex turns the ideas into repo artifacts and working local tools.
- Jeff approves anything public, financial, account-related, or risky.

Do not claim:

- AGI;
- independent personhood;
- guaranteed money-making;
- fully autonomous public posting;
- unbounded trading/funding/shell control.
